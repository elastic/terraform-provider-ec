# Contributing

The canonical contributor guide for the cloud provider is the repo-root
[`CONTRIBUTING.md`](../../CONTRIBUTING.md) — treat it as the source of truth for reporting issues,
the PR flow, dev-environment setup, the pre-submit checklist, commit conventions, and the
`.changelog/{PR}.txt` changelog convention.

For the how-to detail behind it, see [`development-workflow.md`](./development-workflow.md) (make
targets and the change loop), [`testing.md`](./testing.md) (unit + acceptance, required env), and
[`coding-standards.md`](./coding-standards.md).
